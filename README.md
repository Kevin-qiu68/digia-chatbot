开发时的迭代策略：
第一版：实现基础RAG（向量检索+生成），能跑通完整流程。
第二版：添加Rerank，对比效果提升。
第三版：加入Agent能力，处理复杂查询。
第四版：优化UI和异常处理。

每个版本都用git commit保存，方便回退和展示进度。
测试驱动开发：
在VS Code中先写测试用例，再实现功能。准备10-15个标准问题，比如"Digia提供什么服务？"、"如何联系Digia？"、"有哪些成功案例？"等。每次修改后跑一遍测试，记录准确率变化，这些数据可以写进README展示优化效果。
